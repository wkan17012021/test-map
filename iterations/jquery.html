Responsive Image Maps jQuery Plugin

Allows image maps to be used in a responsive design by recalculating the area coordinates to match the actual image size on load and window.resize.

Download the plugin from github

This is a proof of concept for interactive maps on the website.

This image map was created with: https://www.image-map.net/

Other sources of inspiration:

  • https://www.w3schools.com/tags/tag_map.asp. But map tag was part of HTML3 spec and not designed with RWD in mind. Need to come up with a different way of resolving this. The main issue is that the coordinates which define your 'hotspot' areas which you can attach interactivity to, are hard coded absolute values. So, they do not adjust as the dimensions of the image changes.
  • This uses a jQuery plugin but it is not maintained (last PR was 2016). Having said that, someone came up with a plain JS version but I couldn't get this to work: https://github.com/lucakiebel/vanilla-rwdImageMaps. Might be worth revisiting this to refactor code and reduce dependencies.
  • I searched google for ideas but nothing conclusive. I found this blog which uses html and css solution: https://matemarschalko.medium.com/interactive-image-maps-and-closable-popups-with-html-and-css-only-31fe36c70092. It lays the groundwork for this feature. But, the writer conveniently does not mention responsive design, although the post was published in 2021!
  • General SO search led me back to the jQuery solution: https://stackoverflow.com/questions/7844399/responsive-image-map
  • I found this alternative using SVGs: https://thenewcode.com/760/Create-A-Responsive-Imagemap-With-SVG. As the SVG scales, the coordinates should to recalculate based on scale. However, article comments mention it doesn't work with Wordpress and after a similar google search, it appears SVGs are not accepted in Drupal also. At least, not off the bat solution.
  • I did a search on Drupal.org forums to see if anyone else had a similar issue. Turns out a few did and were facing the same problems. Invariably, the outcome was to use the jQuery plugin, or attempt an outdated module install: https://www.drupal.org/forum/general/general-discussion/2021-11-23/image-map-question or https://www.drupal.org/forum/general/general-discussion/2021-03-25/looking-for-a-module-or-other-solution-to-create-a-image
